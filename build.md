# Codex Build Spec: Editorial Frontier Tech Publication

## Core Direction

Build a serious text-first media website for frontier AI, frontier technology, research analysis, and technical news.

This is not a personal blog.
This is not a Substack clone.
This is not an AI startup landing page.
This is not a generic tech news template.

It should feel like:

```txt
Financial newspaper structure
+
research manual clarity
+
independent magazine taste
+
frontier technology subject matter
```

Reference direction:

```txt
Information architecture: Financial Times-style front page hierarchy
Taste / restraint: Making Software-style manual/editorial clarity
Subject matter: frontier AI / research / deep technology
```

Do not copy any publication directly. Use the references only for taste, hierarchy, and interaction patterns.

---

# 1. High-Level Product

The site should support a living publication with:

```txt
News briefs
Research notes
Long essays
Field maps
Topic hubs
Start-here reading paths
Newsletter subscription
Search
Archive
RSS
```

The goal is that I can publish by adding MDX files.

I should not need to redesign pages for every new article.

# 3. Visual Taste Rules

The website must be explicitly non-AI-looking.

Avoid completely:

```txt
gradients
purple/blue AI palettes
glow effects
glassmorphism
animated blobs
3D abstract shapes
generic AI illustrations
stock tech imagery
emoji icons
giant SaaS hero sections
over-rounded cards
fake futuristic dashboards
dark cyberpunk theme
huge empty scroll pages
template-ish spacing
```

Use instead:

```txt
serious typography
warm paper background
near-black text
thin newspaper rules
compact article hierarchy
editorial grids
small labels
careful spacing
plain links
human-written summaries
manual-like topic pages
```

The site should feel designed by someone with taste, not generated by a landing-page template.

# 4. Typography

Typography is the main design system.

Use a newspaper/editorial serif for headings and body.

Recommended stack:

```css
--font-serif: "Newsreader", "Source Serif 4", Georgia, "Times New Roman", Times, serif;
--font-sans: "Inter", "IBM Plex Sans", ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
--font-mono: "IBM Plex Mono", "SFMono-Regular", Consolas, monospace;
```

If using web fonts, use open-source fonts only.

Preferred font choice:

```txt
Headlines: Newsreader or Source Serif 4
Body: Source Serif 4 or Georgia
Interface: Inter or IBM Plex Sans
Code/metadata: IBM Plex Mono
```

Do not use random Google Fonts.

Do not use overly trendy fonts.

Do not use rounded startup fonts.

Do not make the site look like an AI product.

## Body typography

```css
font-family: var(--font-serif);
font-size: 18px;
line-height: 1.65;
letter-spacing: 0;
max-width: 680px;
```

## Headline typography

```css
font-family: var(--font-serif);
font-weight: 700;
letter-spacing: -0.025em;
line-height: 0.98;
```

## Metadata typography

```css
font-family: var(--font-sans);
font-size: 12px;
letter-spacing: 0.04em;
text-transform: uppercase;
```

# 8. Homepage

The homepage is a front page, not a landing page and not a blog feed.

It should feel like a compact, curated newspaper front page.

Route:

```txt
/
```

Homepage structure:

```txt
1. Masthead already visible in global header

2. Lead section:
   - one main featured article
   - 2 secondary stories
   - small latest brief list

3. Latest section:
   - latest briefs / notes / essays
   - compact list, not huge cards

4. Field Maps section:
   - 2 or 3 evergreen major guides

5. Topic rail:
   - AGI
   - World Models
   - Evaluation
   - Compute
   - Interpretability
   - AI Safety
   - India & AI

6. Start Here block:
   - short explanation
   - link to reading paths

7. Newsletter block:
   - restrained bordered signup
```

Homepage should not be endlessly long.

Do not create a giant hero that takes the whole viewport.

The first screen should already show editorial content.

## Homepage layout idea

Desktop:

```txt
------------------------------------------------
Masthead
------------------------------------------------
Main nav
------------------------------------------------

[Main Feature: large headline, dek, metadata]
[Secondary story] [Secondary story]
[Latest briefs list]

------------------------------------------------
Research Notes
- compact list
------------------------------------------------
Field Maps
- 2-column editorial cards
------------------------------------------------
Topics
AGI / World Models / Evaluation / Compute...
------------------------------------------------
Newsletter
------------------------------------------------
```

Use hierarchy like a newspaper.

Use fewer images.

The design should work beautifully with no images.

# 9. Article Type System

There are four main content types.

```txt
Brief
Research Note
Essay
Field Map
```

Each type should have its own listing page and detail template.

All content comes from MDX frontmatter.

Shared frontmatter:

```yaml
title:
subtitle:
description:
date:
updated:
type:
author:
tags:
topics:
summary:
readingTime:
featured:
slug:
related:
sources:
```

The `summary` field should support bullet points.

Example:

```yaml
summary:
  - Energy is a compatibility score, not electricity.
  - EBMs model which states make sense rather than directly predicting outputs.
  - The hard part is training and scaling.
  - Their modern relevance connects to world models and predictive learning.
```

# 10. Briefs

Briefs are short timely news-analysis pieces.

Routes:

```txt
/briefs
/briefs/[slug]
```

Length:

```txt
500-900 words
```

Purpose:

```txt
Keep the site alive.
Explain important recent research/news quickly.
Give technical context without becoming a long essay.
```

Brief detail page structure:

```txt
Title
Subtitle
Metadata: Brief · Date · Reading time · Topics
Summary box
What happened
Why it matters
Technical context
What to watch next
Sources
Read next
Newsletter CTA
```

Brief listing page:

```txt
Page title: Briefs
Description: Short analysis of recent developments in frontier AI and technology.

List:
date
topic label
title
short dek
reading time
```

Briefs should look compact and urgent, but not clickbait.

Use narrow rows and dividers, not giant cards.

# 14. Topic Pages

Routes:

```txt
/topics
/topics/[slug]
```

Topic pages must not be simple tag dumps.

They should be curated mini-magazines / manual chapters.

Initial topics:

```txt
AGI
World Models
Energy-Based Models
Evaluation
Compute
Interpretability
AI Safety
Open Source AI
India & AI
Data-Efficient AI
```

Topic page structure:

```txt
Topic title
Short editorial description

Start Here:
3-5 recommended pieces

Latest:
recent briefs / notes / essays in this topic

Key Questions:
live research questions

Core Concepts:
short glossary-style entries

Further Reading:
external links if available

Newsletter CTA
```

Example:

```txt
World Models

A guide to world models, latent prediction, planning, and their role in AGI research.

Start here:
1. World Models, Explained
2. Why Next-Token Prediction May Not Be Enough
3. JEPA and Latent Prediction

Key questions:
- Can world models scale?
- Do they require embodiment?
- Are they complementary to LLMs?
- What does planning require from a learned model?
```

Topic pages are central to the site.

They make the archive feel intentional.

# 31. Final Taste Checklist

Before considering the design done, check:

```txt
Does it look like a serious publication?
Does it avoid AI/SaaS clichés?
Does it work beautifully without images?
Does the homepage show real editorial hierarchy?
Can a reader quickly find what is new?
Can a reader quickly find where to start?
Do article pages make people read more?
Do topic pages feel curated?
Does the newsletter feel like a return path, not the whole product?
Does it feel human-made?
```

If the answer is no, simplify.

# 33. Hard Instruction

The website should look like a serious media property for frontier research.

It should not look like a portfolio.

It should not look like a startup.

It should not look like a default blog.

It should not look like an AI-generated site.

The design should be quieter, sharper, and more editorial than FT or VentureBeat, while preserving the useful parts of newspaper structure: hierarchy, sections, labels, front-page logic, archive depth, and reader pathways.
